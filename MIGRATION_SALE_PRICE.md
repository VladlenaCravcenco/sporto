# 🔥 Миграция: Добавление системы акций

## Проблема
Вы добавили акционную цену в админке, но товар не отображается в фильтре акций и на главной странице количество акционных товаров = 0.

## Причина
Колонка `sale_price` отсутствует в базе данных Supabase.

---

## ✅ Решение (2 шага)

### Шаг 1: Добавить колонку в Supabase

1. Откройте **Supabase Dashboard** → ваш проект
2. В левом меню: **SQL Editor**
3. Нажмите **New query**
4. Скопируйте и вставьте весь код из файла `/src/database/add_sale_price.sql`:

```sql
ALTER TABLE products 
ADD COLUMN IF NOT EXISTS sale_price NUMERIC(10, 2) DEFAULT NULL;

COMMENT ON COLUMN products.sale_price IS 'Акционная цена. Если NULL — товар не участвует в акции.';
```

5. Нажмите **Run** (или `Ctrl+Enter`)
6. Должно появиться: **Success. No rows returned**

---

### Шаг 2: Очистить кеш в браузере

После выполнения SQL команды:

1. Откройте **DevTools** в браузере (`F12` или `Cmd+Option+I`)
2. Перейдите на вкладку **Application** (Chrome) или **Storage** (Firefox)
3. В левом меню: **Local Storage** → выберите ваш домен
4. Найдите ключи, начинающиеся с `products:` и удалите их:
   - `products:promo_count`
   - `products:count`
   - `products:featured`
   - `products:all`
5. Обновите страницу (`Ctrl+R` или `Cmd+R`)

**Или просто подождите 5 минут** — кеш обновится автоматически.

## 🧪 Проверка

### 1. Проверьте в Supabase Table Editor

1. **Supabase Dashboard** → **Table Editor** → таблица `products`
2. Прокрутите вправо — должна появиться колонка **`sale_price`**
3. Для товара, которому вы добавили акционную цену в админке, проверьте:
   - Колонка `sale_price` должна содержать число (например, `450.00`)
   - Колонка `price` должна содержать старую цену (например, `600.00`)

### 2. Проверьте в админке

1. Откройте админку → **Produse**
2. Найдите товар, которому добавили акционную цену
3. Нажмите на него → должно открыться окно редактирования
4. Поле **"Preț promoțional"** должно быть заполнено

### 3. Проверьте на главной странице

1. Откройте главную страницу (`/`)
2. Найдите карточку с молнией ⚡ **"АКЦИИ"**
3. Количество должно обновиться (например, `5+`)

### 4. Проверьте фильтр в каталоге

1. Откройте каталог (`/catalog`)
2. Нажмите на кнопку с молнией ⚡ в панели категорий (справа от стрелки прокрутки)
3. Кнопка должна стать красной, а в списке должны остаться только товары с акциями
4. Карточки товаров должны показывать:
   - Красный бейдж **"АКЦИЯ"** / **"PROMOȚIE"**
   - Перечёркнутую старую цену
   - Новую цену красным жирным шрифтом

## 📝 Если не работает

### Ошибка: "column does not exist"

Повторите **Шаг 1** — команда `ALTER TABLE` идемпотентна (можно выполнять несколько раз).

### Ошибка: "permission denied"

Убедитесь, что вы используете **service_role key** в настройках проекта, а не **anon key**.

### Данные не обновляются

1. Проверьте **Row Level Security (RLS)** policies:
   ```sql
   SELECT * FROM products WHERE sale_price IS NOT NULL;
   ```
   Если запрос возвращает пустой результат, проверьте политики RLS в Supabase.

2. Откройте **Network** в DevTools и проверьте запросы к Supabase:
   - Должен быть запрос к `/rest/v1/products?...`
   - В ответе должны быть товары с полем `sale_price`
